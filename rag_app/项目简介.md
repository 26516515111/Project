# 项目简介

## 项目名称
船舶装备故障诊断智能问答系统

## 项目定位
面向船舶发动机、液压系统、电气设备等关键装备的故障诊断场景，构建轻量化、可离线运行的检索增强生成（RAG）问答系统，提升故障响应速度与诊断一致性，沉淀可复用的运维知识资产。

## 核心目标
- 将非结构化故障案例与维修手册结构化，形成可检索知识体系
- 实现语义检索与知识图谱检索融合，提高召回与准确率
- 在普通 PC 端离线运行，满足船舶海上场景
- 输出可追溯的维修建议并提供引用依据

## 主要功能
- 混合检索：关键词 BM25 与语义向量检索融合
- 知识图谱：Neo4j 存储故障现象—原因—处理步骤等关系
- 生成式问答：本地 Ollama（qwen2.5vl:7b）生成结构化建议
- 可视化与交互：前端支持历史对话与答案导出
- 可扩展：支持新增设备类型与故障案例快速导入

## 技术路线
- 知识图谱：Neo4j
- NLP 框架：Python + LangChain
- 向量检索：FAISS + Sentence-Transformers
- 生成模型：Ollama 本地推理（qwen2.5vl:7b）
- 前端：Streamlit（可替换为 Flask/HTML）

## 系统流程
1. 文档清洗与分块
2. 向量化与索引构建
3. 关键词+语义混合检索
4. 知识图谱补充关联事实
5. 生成结构化答案并附引用依据

## 预期指标
- 典型问题 Top1 准确率 ≥ 75%
- 单轮响应 ≤ 3 秒
- 内存占用 ≤ 8GB
- 完全离线可运行
